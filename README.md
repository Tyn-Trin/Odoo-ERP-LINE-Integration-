# Odoo LINE Bot — แจ้งเตือน & อนุมัติเอกสาร

**Bot ที่แสดงการแจ้งเตือนจาก Odoo ไปหาผู้ใช้ทาง LINE โดยตรง ทั้งงาน (Activity) ที่ถูก Assign มาใหม่ และเอกสารที่รออนุมัติ — พร้อมให้ Approve/Reject ได้ในแชทเดียว โดยแต่ละคน Login ด้วย Odoo Account ของตัวเอง เห็นเฉพาะข้อมูลและสิทธิ์ของตัวเองเท่านั้น**

ไม่ต้องเปิด Odoo เข้าไปเช็คเองว่ามีงานอะไรมาใหม่ หรือมีอะไรรออนุมัติ — ระบบส่งมาหาเองทันทีที่มีเหตุการณ์เกิดขึ้น ตรงตัวคนที่มีสิทธิ์จริงเท่านั้น

## ปัญหาที่เจอ

- **เอกสารรออนุมัติ** (PO, SO, ใบเบิกจ่าย, ใบขอซื้อ ฯลฯ) มักถูกลืมเพราะไม่มีใครเปิด Odoo เข้าไปเช็ค ทำให้งานค้าง ต้องมีคนโทร/ทักไปตามอีกที ทั้งที่จริงๆ Odoo มีระบบอนุมัติหลายชั้น (Tier Validation) อยู่แล้ว แค่ไม่มีช่องทางแจ้งเตือนที่ผู้อนุมัติเห็นจริงๆ
- **คนที่มีอำนาจอนุมัติส่วนใหญ่ใช้ระบบ ERP ไม่เป็น** — บางคนหน้าที่หลักในองค์กรมีแค่ "อนุมัติ" อย่างเดียว ไม่เคยต้องเปิด Odoo ทำงานอย่างอื่นเลย การจะไปนั่งสอนให้ใช้ Odoo แค่เพื่อกดอนุมัติจึงไม่คุ้มและไม่สมเหตุสมผล
- **ERP ส่วนใหญ่ไม่รองรับมือถือดี** — หน้าจอ Odoo ถูกออกแบบมาสำหรับหน้าจอคอมพิวเตอร์ ใช้บนมือถือได้ลำบาก ทั้งที่ผู้บริหาร/ผู้อนุมัติมักอยู่นอกออฟฟิศ ต้องอนุมัติผ่านมือถือเป็นหลัก
- **งาน (Activity) ที่ถูก Assign มาให้ทำ** เช่น ต้องติดตามลูกค้า ต้องเช็คสินค้า ก็เจอปัญหาแบบเดียวกัน — Notification ใน Odoo มีจริง แต่ไม่มีใครเปิดแอปดู กว่าจะรู้ตัวว่ามีงานมาก็สายไปแล้ว

## ทางแก้

ตั้ง Automation Rule ใน Odoo ไว้ 2 จุด ดักจับ 2 เหตุการณ์แยกกัน แล้วยิง Webhook มาหา Backend ทันที:

1. **เอกสารเข้าสถานะรออนุมัติ** → Backend หาผู้มีสิทธิ์อนุมัติจริง (ตาม Tier Validation reviewer, Multi Approval PIC, หรือ Role ที่ตั้งไว้) แล้วส่ง LINE Flex Message ที่มีรายละเอียดเอกสารครบ (คู่ค้า ยอดเงิน รายการสินค้า) พร้อมปุ่ม Approve/Reject — กดแล้วระบบขอ comment ก่อนยืนยันทุกครั้ง จากนั้นค่อยส่งคำสั่งกลับไปที่ Odoo จริง
2. **มี Activity ใหม่ถูก Assign ให้ User** → Backend หา User คนนั้นที่ผูก LINE ไว้ แล้วส่งข้อความแจ้งเตือนบอกว่างานอะไร เรื่องอะไร กำหนดเมื่อไหร่ — เป็นการ**แจ้งเตือนอย่างเดียว ไม่มีปุ่มกด** เพราะจุดประสงค์แค่ให้รู้ตัวเร็วที่สุด ไม่ใช่ให้ปิดงานผ่าน LINE

## รองรับเอกสารหลายประเภท (ไม่ใช่แค่ PO/SO)

เพราะต่อกับ OCA Tier Validation แบบ generic ทำให้รองรับได้ทุก model ที่เปิด Tier Validation ไว้ เช่น Purchase Order, Sale Order, Invoice, Stock Transfer, Stock Request, Asset Request, Batch Payment, Overtime, Vendor Pricelist, RMA Supplier, Loyalty Program — เพิ่ม model ใหม่แทบไม่ต้องแก้โค้ด

## Tech Stack

| ส่วน | เทคโนโลยี |
|---|---|
| Backend | Python 3.x, FastAPI, Uvicorn |
| ช่องทาง | LINE Messaging API (Flex Message, Webhook + HMAC signature verify) |
| แหล่งข้อมูล ERP | Odoo JSON-RPC + Odoo Automation Rule (Webhook trigger) |
| ระบบอนุมัติ | OCA Tier Validation, Multi Approval |
| Database | PostgreSQL (ผูก User ↔ LINE, Activity Log) |

## Feature เด่น
- **แยก User ตาม Odoo ID จริง** — แต่ละคนที่ผูก LINE ไว้ต้อง Login ด้วย Odoo Account ของตัวเอง (LINE ID ↔ Odoo User ID แบบ 1 ต่อ 1) ไม่ใช่ Bot กลางใช้ Admin Login เดียวปลอมเป็นทุกคน สิทธิ์การเห็นข้อมูล/อนุมัติเอกสารจึงเป็นไปตามสิทธิ์จริงของ Odoo User คนนั้น ไม่ใช่สิทธิ์ของ Bot
- **Push แจ้งเตือนทันที** ทั้งเอกสารรออนุมัติและงานที่ถูก assign — ไม่ใช่ต้องเปิดแอปเข้าไปเช็คเอง
- **รายละเอียดเอกสารครบในแชท** — คู่ค้า ยอดเงิน รายการสินค้า ไม่ต้องเปิด Odoo ดูเพิ่ม
- **ขอ comment ก่อนยืนยันและไม่อนุมัติทุกครั้ง**  — ไม่มีการอนุมัติเงียบๆ โดยไม่ถาม
- **รองรับ Tier Validation หลายชั้น** และ Multi Approval — ส่งหาผู้อนุมัติที่ถูกต้องตามลำดับจริง
- **แจ้งเตือน Activity แยกจากอนุมัติชัดเจน** — แค่บอกให้รู้ ไม่บังคับให้ปิดงานผ่าน LINE (ผู้ใช้ยังต้องไปจัดการต่อใน Odoo)
- **กันแจ้งเตือนซ้ำ (dedup)** เวลา Odoo ยิง Webhook มาหลายรอบสำหรับเหตุการณ์เดียวกัน
- **Audit Log ครบ** — ใครอนุมัติ/reject อะไร เมื่อไหร่ พร้อม comment

## ทำไมถึงทำโปรเจคนี้

จากประสบการณ์ ทั้งเอกสารรออนุมัติและงานที่ถูก assign มักถูกมองข้ามเพราะไม่มีใครเปิด Odoo เช็ค ทั้งที่ระบบข้างในมีครบอยู่แล้ว แค่ขาดช่องทางแจ้งเตือนที่คนเห็นจริง — โดยเฉพาะกับผู้อนุมัติที่ไม่ถนัด ERP และมักทำงานผ่านมือถือ โปรเจคนี้แก้ปัญหานั้นตรงจุด โดยใช้ LINE ที่ทุกคนเปิดอยู่แล้วทั้งวัน
